To implement **Sass** in your project while also using **ESLint** and **Prettier**, you need to ensure that the project setup is aligned with best practices for maintainability, readability, and scalability, especially for organizational-level development. Here's a step-by-step guide to setting it up:

### 1. **Install Sass**
Start by installing **Sass** in your project. Since you're using Vite, React, and TypeScript, you can simply install `sass` for styling.

```bash
# Using Yarn
yarn add sass

# Or using npm
npm install sass
```

Vite will automatically handle Sass imports without additional configuration.

### 2. **ESLint Configuration for Sass (SCSS)**
ESLint focuses on JavaScript/TypeScript linting, but for organizational-level development, you can extend the linting to include **CSS-in-JS** or SCSS files. You can integrate **stylelint** for CSS/SCSS linting, which complements ESLint for styling rules.

#### Install **Stylelint** and Related Plugins:
```bash
# Using Yarn
yarn add stylelint stylelint-config-standard stylelint-scss --dev

# Or using npm
npm install stylelint stylelint-config-standard stylelint-scss --save-dev
```

#### Create `.stylelintrc.js`:
Add a configuration file for **Stylelint** to check your SCSS files:

```javascript
export default {
  extends: ['stylelint-config-standard', 'stylelint-config-recommended-scss'],
  plugins: ['stylelint-scss'],
  rules: {
    indentation: 2,
    'color-hex-case': 'lower',
    'max-nesting-depth': 5,
    'no-duplicate-at-import-rules': true,
  },
  ignoreFiles: ['**/node_modules/**', '**/dist/**'],
};
```

#### Add Stylelint to `lint-staged`:
Update your `lint-staged.config.js` to include SCSS linting with Stylelint:

```javascript
export default {
  '**/*.(ts|tsx)': () => 'npx tsc --noEmit',
  '**/*.(ts|tsx|js|scss)': (filenames) => [
    `npx eslint --quite --fix ${filenames.join(' ')}`,
    `npx prettier --write ${filenames.join(' ')} --ignore-unknown`,
  ],

  '**/*.(md|json)': (filenames) =>
    `npx prettier --write ${filenames.join(' ')} `,
};
```

### 3. **Prettier Configuration for Sass**
Prettier helps keep your code consistently formatted. To work with SCSS in Prettier, make sure you have the correct setup:

#### Install Prettier:
If Prettier isn't already installed, add it:

```bash
# Using Yarn
yarn add prettier --dev

# Or using npm
npm install prettier --save-dev
```

#### Install `prettier-plugin-scss` (optional but recommended):
This plugin ensures that **Prettier** can format your SCSS files.

```bash
# Using Yarn
yarn add prettier-plugin-scss --dev

# Or using npm
npm install prettier-plugin-scss --save-dev
```

#### Update Prettier Configuration:
Make sure your Prettier configuration (e.g., `.prettierrc.js`) includes support for SCSS files:

```javascript
export default {
  semi: true,
  trailingComma: 'all',
  singleQuote: true,
  printWidth: 80,
  tabWidth: 2,
  bracketSpacing: true,
   plugins: [
    'prettier-plugin-scss',  //Curent does not support
  ],
};

```

### 4. **Husky and Pre-commit Hooks**
To enforce linting and formatting before committing code, you should use **Husky** and **lint-staged** together.

#### Husky Installation:
If you haven't installed Husky yet, do so:

```bash
# Using Yarn
yarn add husky --dev

# Or using npm
npm install husky --save-dev
```

Initialize Husky:

```bash
npx husky install
```

Add a pre-commit hook that runs `lint-staged`:

```bash
npx husky add .husky/pre-commit "npx lint-staged"
```

### 5. **Example Project Structure with Sass**
Here's an example of how you can structure your project to ensure scalability and maintainability:

```
src/
  ├── components/
  │   ├── Button/
  │   │   ├── Button.tsx
  │   │   └── Button.module.scss
  ├── styles/
  │   ├── _variables.scss
  │   ├── _mixins.scss
  │   ├── _global.scss
  ├── App.tsx
  └── index.tsx
```

- **Component-level styles**: Place individual component styles as **CSS modules** (e.g., `Button.module.scss`) for better encapsulation.
- **Global styles**: Use a global SCSS folder for variables, mixins, and other shared styles.
- **Importing styles**: You can import global SCSS files in your main `index.tsx` or `App.tsx` file.

### 6. **Organizational Best Practices**
- **CSS Architecture**: Follow methodologies like **BEM (Block Element Modifier)** or **ITCSS** for organizing styles in a scalable way.
- **Linting on CI**: Make sure to run **ESLint**, **Stylelint**, and **Prettier** in your CI pipeline (using tools like **GitHub Actions** or **CircleCI**) to enforce code quality across the team.
- **Consistent Styling**: Using **Tailwind CSS** with SCSS could be beneficial for design consistency and utility-first styling alongside component-specific SCSS for more granular control.
